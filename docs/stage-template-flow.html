Stage Template → Finish Materials Flow

1
Frontend

PM Clicks "Apply Template"

Selects a template & optional start date. POSTs to /api/v1/stages/apply-template

Sends: templateId, projectId, startDate
2
Backend

Validate & Fetch Template Items

Checks user access, confirms no stages exist yet, loads stage_template_items for the template.

3
Database

Create Project Stages

Loops through template items and inserts into project_stages.

For each stage:
• planned_start = previous stage end + 1 day
• planned_end = start + duration_days
• materials_due = start − 7 days
4
Database

Create Material Areas & Items

Fetches material_templates linked to the stage template, then:

a) Creates material_areas (e.g. "Kitchen", "Bathroom")
b) Creates material_items linked to their matching stage
c) All items set to approval_status = 'pending'
5
Frontend

PM Reviews Suggested Materials

A review dialog opens automatically. PM approves or rejects each material via /api/v1/materials/approve-bulk.

• Approved → visible to clients
• Rejected → hidden from clients
